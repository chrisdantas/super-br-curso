CRIANDO PROCESSOS

Antes de incluir um novo processo, o usuário deve se certificar de que ainda não houve o referido cadastro no SUPP, realizando as pesquisas necessárias.

Se o processo já possuir um NUP, deve-se consultar sua existência por meio da pesquisa rápida na barra superior do SUPP.

O QUE É UM NUP?

O Número Único de Protocolo (NUP) é constituído de quinze dígitos principais, mais dois dígitos de verificação, separados em grupos (00000.000000/0000-00).

O primeiro grupo é constituído de cinco dígitos, referentes ao código numérico atribuído a cada unidade protocolizadora e este código identifica o órgão de origem do processo, mantendo-se inalterado.

O segundo grupo é constituído de seis dígitos, separados do primeiro por um ponto e determina o registro sequencial dos processos autuados, devendo este número ser reiniciado a cada ano.

O terceiro grupo, constituído de quatro dígitos, separado do segundo grupo por uma barra, indica o ano de formação do processo.

Se não houver referência a um determinado NUP, ainda assim pode ocorrer de já haver cadastro do processo no sistema, de modo que o usuário deverá realizar uma pesquisa avançada, disponível no painel lateral esquerdo.

Na tela de pesquisa avançada, estão disponíveis diversos campos para que o usuário tente encontrar um cadastro anterior, evitando assim a duplicidade.

Deve-se realizar a pesquisa avançada, ao menos, pelo nome do interessado. Entretanto, quanto mais completa for a pesquisa, mais fidedigna será a base de dados.

Confirmada a inexistência de processo anterior, o usuário deve iniciar o procedimento de criação de um novo processo no SUPP No menu lateral esquerdo o usuário deve selecionar Protocolo > Administrativo:

A página de criação de processo é aberta.

Inicialmente, para começar a criação de um protocolo, é necessário escolher a Unidade Arquivística.

UNIDADE ARQUIVÍSTICA

Existem três tipos de unidade arquivística:

  • Processo: Conjunto das peças apresentadas para servir à instrução e ao julgamento de uma questão
  • Documento Avulso: Um ofício, utilizado para comunicação externa
  • Pasta/Dossiê: Nem tudo é um processo administrativo. Pode ser apenas uma pasta, um depósito de documentos

Após a escolha da Unidade Arquivística mais apropriada, é necessário informar os Dados do Protocolo O sistema precisa saber se o usuário deseja Atribuir Novo Protocolo ou Informar Protocolo Existente. Essa escolha é importante porque o processo em questão pode já existir no sistema.

FIQUE ATENTO:

Ao escolher a opção Pasta/Dossiê como Unidade Arquivística, os Dados do Protocolo são limitados apenas a opção Não Possui Protocolo.

Após os Dados do Protocolo, se encontra a opção de Importar Dados de Processo Existente no Sistema. Essa opção permite a utilização de metadados iguais/semelhantes a um outro NUP cadastrado no sistema. Ao marcar a opção, a caixa de pesquisa de Processo Origem aparece. Ao escolher um processo nesse campo, os metadados do processo são preenchidos, permitindo ainda ajustes por parte do usuário, que pode editar se existir algum metadado diferente.

Ao não escolher a opção de Importar Dados de Processo Existente, é necessário preencher alguns dados para a criação do Protocolo. Os campos marcados com o * são os campos que não podem ficar em branco.

Preenchendo os dados do processo

1. No campo Processo de Trabalho, preenche-se a informação relacionada ao "tipo" do processo, sendo possível escolher entre as opções:

  • Administrativo
  • Consultivo
  • Correicional

É possível preencher o campo e escolher a espécie relacionada ou escolher a opção desejada na e selecionar por meio do botão .

2. No campo Meio, é preenchido o formato do processo, sendo possível escolher entre as opções:

  • Físico
  • Eletrônico
  • Híbrido

3. O campo Classificação deve ser atribuído de acordo com o Plano de Classificação da AGU, o qual é relacionado aos assuntos do CONARQ - Conselho Nacional de Arquivologia. Existem várias opções, aparecendo uma opção de paginação na parte inferior, sendo possível navegar entre os intervalos de Registros por página.

CLASSIFICAÇÃO?

Classificação é o ato ou efeito de analisar e identificar o conteúdo dos documentos arquivísticos e de selecionar a classe sob a qual serão recuperados. Essa classificação é feita a partir de um plano de classificação elaborado pelo órgão ou entidade.

A classificação determina o agrupamento de documentos em unidades menores e o agrupamento destas em unidades maiores, formando o arquivo do órgão ou entidade. Para tanto, deve tomar por base o conteúdo do documento, que reflete a atividade que o gerou e determina o uso da informação nele contida.

A classificação também define a organização física dos documentos, constituindo-se em referencial básico para sua recuperação.

Os objetivos da classificação são:

  • Estabelecer a relação orgânica dos documentos arquivísticos;
  • Assegurar que os documentos sejam identificados de forma consistente ao longo do tempo;
  • Auxiliar a recuperação de todos os documentos arquivísticos relacionados a uma determinada função ou atividade;
  • Possibilitar a avaliação de um grupo de documentos de forma que os documentos associados sejam transferidos, recolhidos ou eliminados em conjunto.


A classificação deve se basear no plano de classificação e envolve os seguintes passos:

  • Identificar a ação que o documento registra;
  • Localizar a ação ou atividade no plano de classificação;
  • Comparar a atividade com a estrutura organizacional para verificar se é apropriada à unidade que gerou o documento;
  • Aplicar a classificação ao documento.

4. É também necessário colocar um Título e um Setor Responsável pelo processo.

5. Os demais campos não são obrigatórios, mas podem conter informações importantes. Na Descrição é possível descrever dados relevantes e observações pertinentes. Já no campo Outro Número é inserido algum número externo que não foi possível aproveitar na criação do NUP.

6. Ao apertar o botão , o NUP é gerado.

O NUP foi gerado, porém são necessários outros procedimentos para cadastramento do processo administrativo, de forma a garantir o seu correto andamento.

Aparece um grid com opções de dados que podem/devem ser preenchidos:

A estrutura das informações no sistema mantem a lógica da estrutura das informações do antigo SAPIENS, onde cada novo evento/movimento é lançado como Tarefa em um processo e o trabalho realizado é a Atividade em uma tarefa.

Por isso, para dar continuidade ao NUP, é necessário a criação de uma tarefa. Porém, existem pré-requisitos para essa etapa.

Por regras negociais, é sempre necessário que se cadastre um assunto e os interessados no processo para poder criar tarefas.

Há um filtro que permite visualizar as tarefas daquele NUP que porventura tenham sido apagadas, uma informação que pode ser gerencialmente importante.
